Grade 12 Capstone Project / Reflections

Bi-weekly Reflection 2

Last Updated: Feb 16, 2026

This week, I have completed the investigation on four devices, the LCD Display, LED Lights, LED Strip, and the PIR Sensor.

The LED Lights are probably the easiest device ever to program. Since there is not much function on the LED Lights, the program can only turn on and off the lights. However, it can be used with a potentiometer and adjusted its brightness later. For now, the program can locate the GPIO Pin that the LED Lights are connected to, turn it on, and turn it off.

The LCD Display is a more problematic device. There was the issue that it was constantly giving errors. On the other hand, the display was not clear enough to see. Although these issues were resolved eventually, programming the LCD Display was still a big task. I had to understand the I2C protocol and get the correct header file from GitHub. After that, I need to learn how to put strings on the LCD Display. It was hard, but I eventually nailed it.

Next, I tried out the LED Strip. The LED Strip was very power-consuming and very bright. At first, when I was assembling it, I couldn’t figure out how to wire the LED Strip because it uses its protocol and has its way of reading and writing the data. I had to look up the device specifications online and understand how the shifting and numbers work to make it turn on and adjust its brightness.

Finally, I programmed the PIR Sensor. The PIR Sensor works very well on its own. However, I found it very hard to stop. Since it triggers the handler after detecting a motion, there is no way to stop the handler. The handler will always be running in the background and will interrupt the program whenever there is another motion detected. Therefore, the workaround for this is to assign a variable to the handler, where the handler only changes the variable but does not print it. Then when the main program needs data from the handler, it can read the variable and use it. This would allow the program to continue running while allowing the handler to do its job.

Overall, this week was a hard week, as it is only the first time I am programming hardware. Over the next two weeks, I will be finishing up the rest of the devices and organizing what each device can do. For now, I am all on top of my deadlines and should be on track to finish the project early next year.
